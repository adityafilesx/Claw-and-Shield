## 🔍 What is ArmorClaw Shield?

**ArmorClaw Shield** is a sophisticated multi-agent AI system that adds a **mandatory safety layer** between an autonomous AI trading agent and real financial markets. Every trade intent — whether generated by a Gemini-powered reasoning engine or a human prompt — must pass through a **20-policy enforcement engine** before execution is allowed.

Think of it as an **AI safety middleware for finance**: the agent proposes, the shield enforces, and only clean, compliant actions reach the broker.

## 🔒 Policy Enforcement Layers

The engine evaluates **20 deterministic policies** grouped into 7 enforcement layers. Every layer must pass — a single violation blocks the trade.

### Layer 0 — Validation & Confidence Gate
| Policy ID | Name | Description |
|---|---|---|
| POL_FAILSAFE | LOW_CONFIDENCE_BLOCK | Blocks any intent with AI confidence < 40% |
| POL_005 | NEGATIVE_QUANTITY_GUARD | Quantity must be a positive integer |
| POL_020 | UNKNOWN_INTENT_BLOCK | Unclassifiable actions blocked by default |

### Layer 1 — Intent Classification
| Policy ID | Name | Description |
|---|---|---|
| POL_015 | SHORT_SELLING_PROHIBITED | Short selling not permitted |
| POL_016 | OPTIONS_TRADING_PROHIBITED | Options trading not authorized |

### Layer 2 — Asset Whitelist
| Policy ID | Name | Allowed Tickers |
|---|---|---|
| POL_004 | TICKER_WHITELIST_ENFORCEMENT | AAPL, MSFT, NVDA, GOOGL, AMZN, TSLA, META, JPM, BAC, SPY, QQQ, GOOG |

### Layer 3 — Quantity & Value Guards
| Policy ID | Name | Threshold |
|---|---|---|
| POL_003 | MAX_SINGLE_TRADE_VALUE | $5,000 per trade |
| POL_012 | MAX_QUANTITY_PER_ORDER | 500 shares per order |
| POL_011 | NO_PENNY_STOCKS | Price must be ≥ $1.00 |

### Layer 4 — Portfolio Risk Limits
| Policy ID | Name | Threshold |
|---|---|---|
| POL_001 | MAX_SINGLE_POSITION_SIZE | 10% of portfolio per position |
| POL_002 | MAX_SECTOR_CONCENTRATION | 30% of portfolio per sector |
| POL_017 | CASH_RESERVE_MINIMUM | 10% cash must remain after trade |
| POL_009 | DAILY_LOSS_CIRCUIT_BREAKER | Halt all trades if portfolio down >5% today |
| POL_010 | SINGLE_POSITION_LOSS_LIMIT | Block new buys if a held position is down >15% |
| POL_019 | OVERSELL_GUARD | Cannot sell more shares than held |

### Layer 5 — Behavioral Rate Limits
| Policy ID | Name | Threshold |
|---|---|---|
| POL_006 | NO_PRE_MARKET_EXECUTION | Trading hours: 9:30 AM – 4:00 PM EST only |
| POL_007 | MAX_DAILY_TRADE_COUNT | 20 trades per calendar day |
| POL_008 | MAX_TRADES_PER_HOUR | 10 trades per rolling 60-minute window |

### Layer 6 — Security & Scope Enforcement
| Policy ID | Name | Description |
|---|---|---|
| POL_013 | NO_EXTERNAL_DATA_EXPORT | HTTP calls only to whitelisted URLs |
| POL_014 | READ_ONLY_RESEARCH_MODE | Research agents cannot execute trades |
| POL_018 | SUB_AGENT_SCOPE_ENFORCEMENT | Sub-agents restricted to capability token |

## 🚀 Getting Started

### Prerequisites

- Python 3.11 or higher
- [Alpaca Markets](https://alpaca.markets) account (paper trading)
- [Google AI Studio](https://ai.google.dev) API key (Gemini)
- [Firecrawl](https://firecrawl.dev) API key (research agent)

### 1. Clone the repository

```bash
git clone https://github.com/adityafilesx/Claw-and-Shield.git
cd Claw-and-Shield/ArmorIQ/armorclaw-shield-2026
```

### 2. Create a virtual environment

```bash
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Configure environment variables

Create a `.env` file in `armorclaw-shield-2026/`:

```env
# Google Gemini
GEMINI_API_KEY=your_gemini_api_key_here

# Alpaca Markets (paper trading)
ALPACA_API_KEY=your_alpaca_api_key_here
ALPACA_SECRET_KEY=your_alpaca_secret_key_here
ALPACA_BASE_URL=https://paper-api.alpaca.markets

# Firecrawl (research agent)
FIRECRAWL_API_KEY=your_firecrawl_api_key_here
```

### 5. Run the demo scenarios

```bash
python -m sandbox.demo_scenarios
```

### 6. Run a full agent cycle

```bash
python -m agent.main_agent
```

### 7. Start the REST API

```bash
uvicorn frontend.api:app --reload --port 8000
```

### Example 1: A compliant trade (ALLOW)

```python
from execution.safe_executor import safe_buy

result = safe_buy("AAPL", qty=3, agent_mode="EXECUTION")
# → {"status": "EXECUTED", "ticker": "AAPL", "qty": 3, ...}
```

### Example 2: Trade blocked by policy (BLOCK)

```python
from execution.safe_executor import safe_buy

# Attempt to buy 300 NVDA shares — would exceed 10% portfolio limit (POL_001)
result = safe_buy("NVDA", qty=300, agent_mode="EXECUTION")
```

```python
from execution.safe_executor import safe_buy

result = safe_buy("TSLA", qty=5, agent_mode="RESEARCH")
```

## ⚙️ Policy Configuration

Policies are defined in `policies/business_rules.yaml` and can be customized without code changes:

```yaml
version: "2.0"
firm: "ArmorClaw Demo Firm"
effective_date: "2026-04-07"

policies:
  - id: POL_003
    name: MAX_SINGLE_TRADE_VALUE
    description: No single trade can exceed $5000 USD
    trigger: [BUY_EQUITY, SELL_EQUITY]
    severity: HIGH
    action: BLOCK
    threshold_usd: 5000          # ← adjust this value

  - id: POL_007
    name: MAX_DAILY_TRADE_COUNT
    description: Maximum 20 trades per calendar day
    trigger: [BUY_EQUITY, SELL_EQUITY]
    severity: HIGH
    action: BLOCK
    threshold_count: 20          # ← adjust this value

whitelist:
  tickers:
    - AAPL
    - MSFT
    - NVDA                       # ← add or remove tickers here
```
